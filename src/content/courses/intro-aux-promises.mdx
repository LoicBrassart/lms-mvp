# Introduction aux Promises

Gérer l'asynchronisme en JavaScript

---

## Pourquoi les Promises ?

Avant les Promises, la gestion de l'asynchronisme reposait sur des **callbacks**.

Le problème : les callbacks imbriqués produisent du code difficile à lire et à maintenir — le **callback hell**.

---

## Anatomie d'une Promise

Une Promise représente la complétion (ou l'échec) d'une opération asynchrone.

```js
const promise = new Promise((resolve, reject) => {
  // opération asynchrone
  resolve("succès");
});
```

---

## Les trois états

- **pending** — en attente du résultat
- **fulfilled** — opération réussie (`resolve`)
- **rejected** — opération échouée (`reject`)

Une Promise ne peut changer d'état qu'une seule fois.
