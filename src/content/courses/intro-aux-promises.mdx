## Pourquoi les Promises ?

Avant les Promises, la gestion de l'asynchronisme en JavaScript reposait sur des callbacks. Le problème : les callbacks imbriqués produisent du code difficile à lire et à
maintenir, surnommé le **callback hell**.

## Anatomie d'une Promise

Une Promise est un objet qui représente la complétion (ou l'échec) d'une opération
asynchrone.

```js
const promise = new Promise((resolve, reject) => {
  // opération asynchrone
  resolve("succès");
});
```

Elle peut être dans trois états : pending, fulfilled, ou rejected.
